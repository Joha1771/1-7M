Конспект: Redux

Redux — это библиотека для управления состоянием (state) в приложении.
Все данные хранятся в одном месте, и их можно изменять только через специальные механизмы.

Основные части:
Store — общее хранилище, в котором находятся все данные приложения.
Action — объект, который описывает, какое действие нужно выполнить.
Содержит:
type — тип действия
payload — данные (если нужны)

Пример:

{ type: "ADD_PRODUCT", payload: "Apple" }


Принцип работы
1. Отправляется action
2. Reducer обрабатывает его
3. Обновляется state в store
4. Интерфейс автоматически обновляется


Итог
Redux — это способ хранить и изменять данные приложения через единый источник и понятную структуру (store → action → reducer).


Зачем используется
Redux нужен, когда:
1. в приложении много компонентов
2. данные должны использоваться в разных местах
3. Он помогает централизованно управлять hooks и избегать путаницы.
